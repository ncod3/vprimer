# V-primer 利用方法 (日本語)

vprimer [options]

## 必須項目（２つ）

<dl>
<dt>
--ref リファレンスFasta
</dt>
<dd>
<p><p>
VCFファイル作成に用いたbamファイルが参照しているリファレンスFastaを指定する。プレーンテキストでもgzip圧縮形式でも可。
</p></p>
</dd>
</dl>



<dl>
<dt>
--vcf VCFファイル
</dt>
<dd>
<p><p>
各サンプルのバリアントが収納されたVCFファイルを指定する。
</p></p>
</dd>
</dl>


## 選択項目

<dl>
<dt>
--out_dir 結果出力ディレクトリ名
</dt>
<dd>
<p><p>
結果を出力するディレクトリ名を指定する。デフォルトは「out_vprimer」。
</p></p>
</dd>
</dl>



<dl>
<dt>
--show_samples
</dt>
<dd>
<p><p>
解析の事前処理として、リファレンスFasta内に収められたcontig名とそのサイズを表示、その後、VCF内に収められたサンプル名を表示して終了する。システム内ではサンプル名を３種類で指定可能である。VCFファイルに書かれたサンプル名そのもの (fullname)、'/' で区切られたPATHを取り除いた名前 (basename)、basenameから拡張子.bamを取り除いた名前 (nickname)である。この情報は、refsの中の _sample_name.txt という拡張子の付いたファイルに保存されており、このファイルの nickname はユーザが編集でき、その名前で使用できる。
</p></p>
</dd>
</dl>



<dl>
<dt>
--show_fasta
</dt>
<dd>
<p><p>
解析の事前処理として、fasta内に収められたcontig名とそのサイズを表示して終了する。
</p></p>
</dd>
</dl>



<dl>
<dt>
--a_sample { サンプル名 [ サンプル名 ... ] }<br>
--b_sample { サンプル名 [ サンプル名 ... ] }
</dt>
<dd>
<p><p>
ユーザが解析したいグループを２つに分ける時の、各グループのサンプル群。片側に「ref」を指定すると、リファレンスFastaを単一のサンプルとして扱うことができる。サンプル名のセパレータは、ホワイトスペースか、カンマ。
</p></p>
</dd>
</dl>



<dl>
<dt>
--auto_group { サンプル名 [ サンプル名 ... ] }
</dt>
<dd>
<p><p>
解析するグループを固定で分けず、genotypeで自動的に分ける時に使用するサンプル群を指定する。VCFファイル内のすべてのサンプルを示す「all」を指定可能。最終的なプライマー情報には、サンプル群がどのように分けられたかが記述される。
</p></p>
</dd>
</dl>


<!-- ここからしたがまだ -->

<dl>
<dt>
--pick_mode { indel | caps | snp }
</dt>
<dd>
<p><p>
３種類の解析モードを指定する。indelとcapsは並列して指定可能だが、snpは単独でしか指定できない。デフォルトは、「indel」。
</p></p>
</dd>
</dl>



<dl>
<dt>
--target { scope := chrom_name | chrom_name:range_str }
</dt>
<dd>
<p><p>
マーカーを取得するリファレンスfasta上の範囲を指定する。「all」と指定すると全染色体が対象となる。個別の範囲指定は、染色体名、または染色体上の範囲 (chrom_name:from-to) を列挙する。
</p></p>
</dd>
</dl>



<dl>
<dt>
--indel_size 最小サイズ-最大サイズ
</dt>
<dd>
<p><p>
indel解析モードの際に、indel長(対象とする２つのバリアントの長さの差)の範囲を指定する。最大サイズを越える差は解析対象外となる。最小サイズ未満の差はindel解析モードでは対象外となるが、caps解析モードでは解析対象となる。したがって解析モードでSNPのみを対象にしたい場合は、最小サイズを１として設定することにより、長さの差が０のバリアントのみを caps の解析対象とすることができる。デフォルトは、「20-200」。
</p></p>
</dd>
</dl>



<dl>
<dt>
--product_size 最小サイズ-最大サイズ
</dt>
<dd>
<p><p>
primer3に指示するプライマーのプロダクトサイズの範囲を指定する。複数の範囲群を一度に指定することはできない。デフォルトは、「200-500」。
</p></p>
</dd>
</dl>



<dl>
<dt>
--homo_only
</dt>
<dd>
<p><p>
解析するバリアントを、ホモ接合にのみに限定する。00/11 などのgenotypeが対象となり、00/01 は対象外となる。
</p></p>
</dd>
</dl>



<dl>
<dt>
--enzyme { enzyme_name [ enzyme_name ... ]}
</dt>
<dd>
<p><p>
caps解析モードで使用する制限酵素名を羅列する。使用可能な制限酵素名は、refs/caps_enzyme_name_list.whole_enzyme.txt に記述されているものを指定できる。
</p></p>
</dd>
</dl>



<dl>
<dt>
--enzyme_file { enzyme_file [ enzyme_file ] }
</dt>
<dd>
<p><p>
caps解析モードで使用する制限酵素名を羅列したファイルを指定する。複数のファイルを指定可。使用可能な制限酵素名は、refs/caps_enzyme_name_list.whole_enzyme.txt に記述されているものを指定でき、１行に１つ制限酵素名を記述する。２カラム目以降に、制限酵素名の情報が書かれていても無視される。
</p></p>
</dd>
</dl>



<dl>
<dt>
--p3_normal p3パラメータ記述ファイル
</dt>
<dd>
<p><p>
indelおよびcaps解析モードにおいて、Primer3 に与えるパラメータが書かれたファイルを指定する。デフォルトは refs/p3_normal.txt で、このファイルを直接編集しても良い。当初の内容は refs/p3_normal.txt.original として保存されている。
</p></p>
</dd>
</dl>



<dl>
<dt>
--p3_amplicon p3パラメータ記述ファイル
</dt>
<dd>
<p><p>
snp解析モードにおいて、Primer3に与えるパラメータが書かれたファイルを指定する。デフォルトは refs/p3_amplicon.txt で、このファイルを直接編集しても良い。当初の内容は refs/p3_amplicon.txt.original として保存されている。
</p></p>
</dd>
</dl>



<dl>
<dt>
--amplicon_param タグおよびTMしきい値
</dt>
<dd>
<p><p>
snp解析モードで、設計されたプライマーに接着するタグを含めてのヘアピン構造チェックと、ホモ/ヘテロダイマーチェックのために必要な情報を指定する。デフォルトは、「ACACTGACGACATGGTTCTACA,TACGGTAGCAGAGACTTGGTCT,45,40」である。カンマで区切られた４つの値で構成され、１、２の配列はそれぞれフォワード／リバースタグ配列で、３はヘアピン構造のTMしきい値(45)、４はダイマー構造のTMしきい値(40)である。
</p></p>
</dd>
</dl>

<dl>
<dt>
--min_max_depth 最低デプス-最高デプス
</dt>
<dd>
<p><p>
サンプルに対応するbamファイルを参照可能ならば、プライマー設計時に各サンプルのデプスが少ないところ、多すぎるところを避けることが可能である。その場合の、最低デプス、最高デプスを指定する。デフォルトは「8-300」。
</p></p>
</dd>
</dl>



<dl>
<dt>
--bam_table
</dt>
<dd>
<p><p>
プライマー設計時のbamのデプス考慮のための、サンプル名とbamファイルの対応表を指定する。最初のカラムはサンプル名、次のカラムは、bamファイルもしくはbedファイルの位置である。refs の下に、拡張子 _sample_bam_table.txt として、サンプル名が書かれた対応表のテンプレートが準備されているので、２カラム目にファイル名を記述することで使用できる。また、一度bamから解析が終わると、refs/bedの下に {サンプル名}.bam.m8_x300.bb.bed のようなbedファイルができるので、このbedファイルを参照することも可能である。
</p></p>
</dd>
</dl>


<dl>
<dt>
--progress フェイズ名
</dt>
<dd>
<p><p>
vprimerの実行フェイズは以下の８つに分かれている。prepare, variant, marker, primer, formfail, formpass, snpfilter, chkhdimer である。ここでは、開始したいフェイズ名を指定することで、それ以前の結果をもとにして、新しく解析をやり直すことができる。
</p></p>
</dd>
</dl>



<dl>
<dt>
--stop フェイズ名
</dt>
<dd>
<p><p>
--progress と同様、停止するフェイズ名を指定する。
</p></p>
</dd>
</dl>



<dl>
<dt>
--thread 使用CPU数
</dt>
<dd>
<p><p>
使用するCPU数を指定する。デフォルトは２。
</p></p>
</dd>
</dl>



<dl>
<dt>
--ini_file
</dt>
